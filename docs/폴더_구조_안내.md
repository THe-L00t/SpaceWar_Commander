# 📁 SpaceWar Commander 폴더 구조 안내

이 문서는 프로젝트 폴더가 **어떤 역할로 나뉘어 있는지** 쉽게 정리한 가이드입니다.
"이 파일은 어디에 넣지?" 싶을 때 여기를 보면 됩니다.

> 기술 스택: **DirectX 12 + DXR(레이트레이싱)** 기반 커스텀 C++ 엔진
> 미들웨어: **FMOD**(사운드), **Assimp**(모델 임포트/익스포트)
> 렌더링 SDK: **NRD**(레이트레이싱 디노이저), **DLSS**(AI 업스케일링) — 둘 다 NVIDIA

---

## 최상위 구조 (딱 2개)

```
SpaceWar_Commander/
├── docs/        📄 기획서·문서·일정  ← 지금 작성 중
└── Project/     💻 실제 게임 프로젝트 (코드·셰이더·리소스·라이브러리)
```

- **`docs/`** : 기획과 문서. 지금 미리 정리해두는 영역입니다.
- **`Project/`** : 게임 본체. **비어 있으며, Visual Studio에서 프로젝트를 생성해 직접 관리합니다.**
  (아래 "Project 내부 구성 계획"은 그때 참고할 **설계도**일 뿐, 미리 폴더를 만들 필요는 없습니다)

---

## 📄 docs/ — 문서

졸업작품에서 **평가 비중이 큰** 부분입니다. 기획과 설계를 여기에 모아둡니다.

| 하위 폴더 | 넣는 것 |
|-----------|---------|
| `기획안/` | 게임 기획서(GDD), 세계관·스토리, 게임플레이, UI/UX 기획 |
| `기술문서/` | 아키텍처, 렌더링 파이프라인(DX12/DXR) 설계, 빌드 가이드 |
| `일정/` | 마일스톤, 주차별 개발 로그 |
| `references/` | 참고 논문, 레퍼런스 이미지, 자료 |

## 💻 Project/ — 게임 프로젝트

**현재 비어 있습니다.** Visual Studio에서 직접 프로젝트(솔루션)를 생성하고, 내부 구조도 그 안에서 관리합니다.

> 💡 Visual Studio 솔루션(`.sln`)과 프로젝트 파일을 이 `Project/` 폴더 안에 만드세요.
> 빌드 결과물(중간 파일, 실행파일 등)은 VS가 자동으로 만들어 줍니다.

## 🗂 Project 내부 구성 계획 (VS 작업 시 참고 설계도)

아래는 `Project/`를 **어떻게 구성하면 좋을지에 대한 참고 설계도**입니다.
지금 폴더를 만들 필요는 없고, Visual Studio에서 프로젝트를 관리하며 참고하세요.

### 💻 src/ — 소스코드
C++ 코드를 **"엔진"과 "게임"으로 나눠서** 관리합니다.

- **`Engine/`** — 어떤 게임에도 쓸 수 있는 재사용 가능한 부분
  - `Core/` : 앱 실행, 창(Window), 타이머, 로그
  - `Graphics/D3D12/` : DX12 렌더링 (Device, CommandQueue, SwapChain, PSO)
  - `Graphics/DXR/` : 레이트레이싱 (가속구조 BLAS/TLAS, RT 파이프라인, SBT)
  - `Graphics/Denoise/` : 레이트레이싱 결과 노이즈 제거 (NRD 래퍼)
  - `Graphics/Upscale/` : 저해상도 → 고해상도 AI 업스케일링 (DLSS 래퍼)
  - `Resource/` : 모델·텍스처 로딩 (Assimp 래퍼)
  - `Audio/` : 사운드 재생 (FMOD 래퍼)
  - `Scene/` : 씬 그래프, 카메라, 엔티티
  - `Input/` : 키보드·마우스 입력
  - `Math/` : 벡터·행렬 등 수학 유틸
- **`Game/`** — "우주전 커맨더" 게임 고유 로직
  - `Gameplay/` : 게임 규칙, 전투 흐름
  - `Entities/` : 함선, 무기, AI 등 등장 요소
  - `UI/` : 게임 화면 UI

> 💡 엔진과 게임을 나누면 발표·심사 때 구조 설명이 훨씬 깔끔합니다.

> 🔎 **렌더링 흐름 (참고):** DXR(레이트레이싱) → **Denoise(NRD)** 로 노이즈 제거 →
> **Upscale(DLSS)** 로 고해상도 확대. 즉 저해상도로 레이트레이싱한 뒤 디노이즈·업스케일해서
> 성능과 화질을 동시에 잡는 구조입니다. (NRD·DLSS 모두 모션벡터가 필요하니 `D3D12`에서 함께 준비)

### 🎨 shaders/ — HLSL 셰이더 (원본)
GPU에서 도는 셰이더 **소스코드**를 둡니다. (컴파일 결과물은 빌드 폴더로 감)

- `raster/` : 일반 래스터 셰이더 (Vertex/Pixel)
- `raytracing/` : DXR 셰이더 (raygen, closesthit, miss, anyhit)
- `compute/` : 컴퓨트 셰이더
- `common/` : 여러 셰이더가 공유하는 include 파일(.hlsli)

### 📦 assets/ — 게임 리소스
게임 실행 중에 불러오는 실제 콘텐츠를 둡니다.

| 하위 폴더 | 넣는 것 |
|-----------|---------|
| `models/` | 3D 모델 (fbx / gltf / obj — Assimp가 읽음) |
| `textures/` | 텍스처 이미지 |
| `materials/` | 재질 정보 |
| `audio/` | 사운드 (FMOD `.bank`, 원본 사운드) |
| `fonts/` | 폰트 |
| `scenes/` | 레벨·씬 데이터 |
| `config/` | 게임 설정 파일(json 등) |

### 🔌 external/ — 외부 라이브러리
직접 만들지 않고 **가져다 쓰는** 라이브러리(SDK)를 둡니다.
게임 소스에는 안 들어가지만 링크해서 사용합니다.

- `fmod/` : FMOD 사운드 SDK
- `assimp/` : 모델 임포트/익스포트 라이브러리
- `nrd/` : **NRD** (NVIDIA Real-Time Denoisers) — 레이트레이싱 노이즈 제거
- `dlss/` : **DLSS** (NVIDIA NGX SDK) — AI 업스케일링
- `imgui/` : 디버그용 UI (사용 시)

> ⚠️ **중요:** `external/README.md`에 각 라이브러리의 **버전·다운로드 링크·라이선스**를 꼭 기록하세요.
> 나중에 환경 재설치나 제출 때 필요하고, 특히 FMOD·DLSS는 라이선스 고지 의무가 있습니다.

> 💡 **NRD·DLSS는 NVIDIA 전용**입니다. 발표·심사 PC가 NVIDIA RTX GPU가 아닐 수 있으니,
> "DLSS/NRD를 끈 상태에서도 돌아가는 기본 경로"를 함께 만들어두는 걸 권장합니다.

### 🛠 tools/ — 빌드·변환 도구
개발을 돕는 스크립트를 둡니다.

- `shader_compile/` : dxc로 셰이더를 컴파일하는 스크립트
- `asset_pipeline/` : Assimp로 모델을 변환하는 등의 도구
